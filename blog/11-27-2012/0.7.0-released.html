0.7.0 Released.

I have just bundled and pushed a pretty lame 0.7.0 release to github. Why so lame? Well, this release was mostly pushed to get the *current* docs to line up with the *dev* docs. So, more of a documentation up than anything else. This finally sees the site move to Gen for generation, the source is available here.

Having said that however, there is a lot of work in this release. It just doesn't really close any of the outstanding issues in the 0.7.0 milestone. These will likely be moved into a 0.8.0 or maybe in fact a 0.10.0 release. See the Changelog

Moving forward — 0.10.0

The next version may indeed skip a few beats. Why? Some major refactoring.

For a while now there have been a few fairly major niggles eating away with me. These things need to be fixed before the framework can really start to move forward. These include but are not really limited to:

  • Directory structure. - The current structure and naming convention is terrible.
  • The test suite - Needs a serious overhaul
  • IO - I've been thinking for along while now that this could use the Symfony\HttpFoundation component.
  • The Bootstrap Filter Chain - Currently a lot more complex than it needs to be. This is somewhat of a performance bottleneck.

None of these are small tasks, and in some cases, things will be completely rewritten.

Anyway, for now, I'm just glad to see some movement in the repos.

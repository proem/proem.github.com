Essential Framework Components?

Most frameworks these days are split into parts, the kernel and then the rest of the components. It's fair to say that as a kernel, Proem is getting pretty close to being feature complete. However, it is these components that users will miss if they where to start using it now.

So, bearing in mind that Proem really is striving to be a minimal framework, I'd like to pose the question: What components would you consider as being an essential part of a framework?

I already have a few ideas in mind, and even have plans on developing some of these in the 0.7.0 milestone.

  • A PHP Rendering View layer
  • Access Control Lists
  • Session Handling
  • Config Management

I would however love some feedback on this subject. I will likely develop most of these components as Plugins, but still, some of these plugins should ship with the framework out of the box. Thoughts?

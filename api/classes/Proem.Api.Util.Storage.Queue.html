Priority queue implementation.

Implements the \IteratorAggregate and \Countable interfaces.

Internally we use SplPriorityQueues to implement the priority parts. Wrapping the SplPriorityQueue within a IterattorAggregate means however that we can rewind and play the queue again if required. This cannot be achieved with the SplPRiorityQueue alone because it is a stack and as such items are lost as they are iterated over.

Overview

The Proem framework aims to be a lightweight, simple & fast MVC implementation built on top of PHP5.4

The overall design has been kicking around my head for a number of years, slowly becoming more and more refined over time. This is actually the third iteration of the idea, with the previous attempt being stalled here.

The current idea will see the framework become Event driven, with the Bootstrap process executing a filter chain to instantiate the bulk of the components required to build an application. During the execution of each step in the filter chain more events will be exposed and these events will be easy to subscribe / listen too.

This should see the bootstrap process become very easy to extend and customise. I will begin documentation as I get components completed.

We currently have this website, http://proemframework.org, an irc channel on #proem irc.freenode.net, and our issue tracker and code repositories on GitHub.

Please be aware that while this documentation is accurate, Proem is a fast moving target at this point in time.

Proem is still very much in early pre-alpha stages and is not currently usable. There is no Controller or View components currently in place.
